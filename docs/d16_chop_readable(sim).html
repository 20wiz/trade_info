📖 용어 풀이 — In-sample vs Out-of-sample
In-sample: "내가 가진 데이터로 최고로 맞춘 결과". 마치 시험문제와 답을 같이 보고 만점 받는 것. 보통 너무 좋아 보임.
Out-of-sample (OOS): "다른 데이터로 시험". 본 적 없는 데이터에서 같은 규칙이 작동하면 진짜 edge 가 있는 것.

Task 5 가 in-sample 에서 5y +228% 까지 만들었지만, "다른 해로 시험" 해 보니 거의 다 fragile 했음. 우리는 진짜 OOS robust 한 규칙을 찾아야 했음.

2. 처음 가설 → 검증 → 정반대 발견

1처음 가설: "2023 = chop 시기"
2023년이 ETH 가 위아래 흔들리는 chop (range-bound) 시기였다고 추정. 그렇다면 "시장이 chop 일 때 진입 금지" 라는 규칙이 통할 것.

측정 도구:
  • Choppiness Index (CI): 100 에 가까울수록 chop, 0 에 가까울수록 강한 트렌드. 1시간봉 (F11), 4시간봉 (F12) 두 가지.
  • ADX (Average Directional Index): 트렌드 강도. 낮으면 weak/chop, 높으면 strong trend. 일봉 (F13), 4시간봉 (F14), 1시간봉 (F15).
  • F16: 일봉 ADX < 25 AND 4시간봉 ADX < 25 (= 두 시간대 모두 chop) 이면 차단.
📖 용어 풀이 — ADX 한 줄 설명
ADX(14) 는 "지난 14봉 동안 가격이 한 방향으로 얼마나 강하게 움직였는가" 를 0~100 으로 보여주는 지표.
ADX 20 이하: 트렌드가 약함 (옆으로 가는 시장 = chop). ADX 25~40: 명확한 트렌드. ADX 40 이상: 매우 강한 트렌드 (또는 과열).
ETH 의 일봉 ADX 가 30 이라는 건 "ETH 가 일봉 기준으로 강한 추세 안에 있다" 라는 의미.

  • 2023 ETH +91% recovery 는 일봉으로는 강한 직진 상승. 그러나 그 안에서 5분봉 long entry 가 "이미 멀리 와버린 상태" 에 들어가 mean-reversion 에 잡힘.
  • 2024 winners 의 ADX 18 = mild bull. 5분봉 entry 가 mean-reversion 으로 작동하는 환경.
  • 핵심: 일봉 ADX > 30 시기에는 진입 차단.

3. 진짜 검증: "다른 해로 시험하기" (Walk-forward)

📖 용어 풀이 — Walk-forward (보행자 검증)
단순 in-sample sweep 의 함정은 "수많은 후보 중 우연히 좋은 것 1개" 를 진짜라고 착각하는 것.
Walk-forward: 데이터를 시간순으로 잘라서 train (학습) → test (검증) 으로 분리. 예) 2024-2026 데이터로 best threshold 를 찾은 다음, 그 threshold 를 2023 데이터에 적용해서 진짜 작동하는지 보는 것.
본 작업에서는 leave-one-year-out: 4년치로 best threshold 학습 → 1년치로 검증 → 4년 모두 돌아가며 반복.

3.2 왜 F13 만 다른가 — 직관적 설명

다른 features 가 한쪽 해에 fit 한 이유

F1 (EMA200 거리), F8 (BTC-ETH 7d 차이), F11 (1h Choppiness) 등은 그 해의 시장 character 자체에 묶여있음. 2024 의 EMA200 거리 분포는 2023 과 다르고, 2024 fits 한 threshold 는 2023 에서 무력.

F13 (1d ADX) 이 robust 한 이유 (추정)

ADX 는 "트렌드의 강도" 라는 universal property 측정. ETH 의 일봉 추세가 강할 때 (33+) 5분봉 mean-reversion 진입이 잡혔다는 패턴이 시장 cycle 과 무관하게 작동. 2023 과 2025 는 다른 cycle 이지만 같은 규칙이 fire.

4.2 5년 시뮬레이션 결과 (50% staking compound)

📖 용어 풀이 — Compound (복리)
Compound = (1 + r1) × (1 + r2) × ... × (1 + rn) − 1. 즉 매 trade 의 결과가 누적되어서 다음 trade 의 stake 에도 반영.
우리 봇은 wallet 의 50% 를 stake 로 사용 (config 의 tradable_balance_ratio=0.5). 그래서 정확한 wallet 변화는 (1 + 0.5 × r1) × (1 + 0.5 × r2) ... 입니다.
Freqtrade 가 표기하는 "Total profit %" 와 일치하도록 동일 공식 사용 (2024 D14.A 의 +52.13% 와 numerical match 검증함).

5. 아직 부족한 부분 (정직하게)

⚠️ 1. 실제 freqtrade backtest 아직 못 돌렸음

freqtrade 가 backtest 시 ccxt async 로 Binance markets 정보를 받아오는데, 현재 환경에서 aiohttp + Binance 가 작동 안 함 (sync ccxt 와 직접 urllib 는 OK).

우회: D14.A 의 trades.json 데이터에서 entry timestamp 별로 새 1d ADX 값을 직접 계산해 "차단 시뮬레이션". 127/127 trade 의 c5 값이 분석 csv 와 일치 확인 → indicator 정확성은 보장.

그러나 freqtrade 의 실제 wallet sequence 효과 (trade 차단 시 다음 trade 의 timing 변화) 는 미반영 → 실제는 ±5~10pp 차이 가능.

⚠️ 2. 2022 / 2026 의 sample 이 거의 없음

2022: 1건만 의 trade. 통계적 신뢰도 매우 낮음.
2026: 0건 (D14.A 이 모든 진입 차단 — binary blocking 문제 그대로 유지).

즉 우리가 "5년 robust" 라고 부르는 건 실질적으로 2023+2024+2025 의 3년 walk-forward. 정직하게 말하면 "3년 OOS robust".

⚠️ 3. Strict gates (G1, G5) 는 여전히 fail

엄격한 채택 기준 (모든 해 양수, sign agreement 5/5) 으로는 여전히 미달. 2023 −23% 는 개선됐지만 여전히 음수.

따라서 채택 결정은 "엄격 기준 vs compound 기준" 의 trade-off — 운영자 결정 사항.

⚠️ 4. In-sample overfitting 의 잔여 가능성

F13 threshold 30 / 33 자체는 in-sample 분포 보고 결정한 값. walk-forward 가 robust 함을 보여주긴 했지만, 완전 future OOS (예: 2027 년) 에서 같은 threshold 가 작동할지는 보장 못 함.

→ 실제 dry-run 시 6개월 shutdown protocol (월별 NEG 3회 연속 시 자동 정지) 필수.

6.2 환경 이슈 해결 가이드 (사용자 작업)

🔧 freqtrade backtest 가 실행 안 되는 이유
freqtrade backtesting 명령이 ccxt.async_support.binance.load_markets() 단계에서 ExchangeNotAvailable 에러로 죽음.
직접 검증:
  • ✓ urllib 로 직접 Binance API 호출: 200 OK
  • ✓ ccxt.binance (sync) 로 load_markets: 4363 markets 정상
  • ✗ ccxt.async_support.binance (async, aiohttp): ExchangeNotAvailable
→ freqtrade 가 async 만 사용하므로 backtest 진행 불가.

옵션 A — Linux/Docker (권장)

Linux/WSL 또는 Docker container 에서 동일 명령 실행. aiohttp 환경이 다르면 정상 작동할 가능성 높음.

옵션 B — VPN

한국 IP 가 일시적 차단됐을 가능성. VPN으로 다른 region IP 사용 후 시도.
