1. composite_h 전략 설명

1.1 전체 구조

composite_h 는 Long_v14_5m_iter2391 계열의 가장 깊은 자손이며, 여러 entry 필터를 OR composite 형태로 누적한 long-only 전략입니다. 진입 신호 (5분/4시간 mean-reversion + ADX trend + funding) 이 트리거되면, 다음의 "bear / overextension 차단 필터" 들이 차례로 검사되어 하나라도 발동하면 진입 차단:

차단 필터의미구현
BTC EMA bear BTC 1d EMA20 < EMA60 일 때 (시장 전반 베어) D12 v2: btc_filter 부모에서 처리
ETH 30d drawdown (c2) ETH 가격이 30일 최고가 대비 −12% 이상 하락 D14.A composite_a 추가 layer
ETH 1d ADX > 28 (c5) 🎯 일봉 트렌드 강도가 매우 높을 때 (overextended) D16.H composite_h 추가 layer (이번 작업의 핵심)

1.2 D16.H 의 가설

가설: 2023 의 catastrophic loss 시기 (ETH +91% recovery 인데도 long entry 53/53 모두 손실) 는 일봉 기준 "강한 직진 상승의 막바지 = overextension" 시기였다. 일봉 ADX(14) 가 28 을 넘는 시기에 5분봉 long entry 를 차단하면 이런 chop-after-overextension 손실을 피할 수 있다.

1.3 진입 logic 사다리

1 5m bar 에서 regime classifier 가 bull_highvol 로 분류
2 4h ADX trend ≥ 25 (isolation gate)
3 Mean-reversion signal (RSI / BB / disp) 조건
4 Funding 정상 (양의 funding)

→ 위 4단계 통과 시 enter_long = 1 후보. 이후 차단 필터 3개 (BTC bear OR ETH dd OR ETH 1d ADX>28) 가 하나라도 fire 하면 차단.

1.4 5m vs 4h timeframe 의 영향

요소5m (원본)4h (이번 백테스트)
Entry candle매 5분매 4시간
Detail candle(없음)1h (entry/exit fine-grain)
RSI / BB / disp 계산5m bars 위에서4h bars 위에서
ATR_1h_at_arm1h informative 데이터4h 기준으로 stop 계산이 거칠어짐
F13 (c5) 1d ADX5m → 1d resample4h → 1d resample (동일 결과)
Signal density1년 약 30~60건1년 약 0~1건

4. 왜 4h 에서는 안 통하나 — 진단

composite_h 의 모든 진입 신호 logic (RSI, BB, disp, ATR, MACD, ADX 등) 은 5분봉 위에서 calibrate 됐습니다. 4h 로 timeframe 을 옮기면:

4.1 Signal density 의 붕괴

요소5m4h비율
연간 캔들 수~105,000~2,200×0.02
연간 entry 후보 (D14.A)~30~0~3×0.05
composite_h 통과 entry12~210~1×0.05

4.2 hardcoded threshold mismatch

RSI / BB-disp 의 threshold (예: RSI < 30, BB-width < 1.5%) 가 5m bar 의 분포 기준으로 잡혀있음.
4h bar 는 변동성이 훨씬 크고 분포가 다름 → 같은 threshold 가 4h 에서는 거의 발동하지 않음.

4.3 4h 특정 indicator 의 의미 변화

5m 전략은 ATR_1h_at_arm, 4h regime 등의 multi-timeframe informative 를 사용. 4h 가 primary timeframe 이 되면 같은 indicator 가 다른 컨텍스트로 해석되어 entry 조건이 비정합.

4.4 결과 해석

4h 에서 발생한 2 trades 는 모두 stop-loss 로 종료. 이는 4h 위의 signal logic 이 "정상" 입력으로 인식하지 않고, 가뭄 끝에 우연히 통과한 caustom edge case임을 보여줌. 만약 이 전략을 4h timeframe 에서 운영하려면 모든 threshold 를 4h 분포 기준으로 재-calibrate 해야 함 (사실상 새 전략 개발).
